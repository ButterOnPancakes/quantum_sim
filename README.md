# quantum_sim

A **state‑vector quantum computer simulator** written in C. It implements a full gate library, high‑level circuits (Quantum Fourier Transform, phase estimation, order finding) and a complete **Shor's algorithm** implementation for integer factorisation.

## What It Does
`quantum_sim` simulates quantum circuits on classical hardware using a **state‑vector** representation.  The full quantum state of *n* qubits is stored as a vector of \(2^n\) complex amplitudes, and each gate updates this vector in place.

---

## Features
| Feature | Details |
|---|---|
| **Quantum register management** | Create, fuse, measure, and free registers |
| **Classical register management** | Store measurement outcomes |
| **Single‑qubit gates** | H, X, Y, Z, phase‑shift (Rz) |
| **Two‑qubit gates** | CNOT, controlled‑rotation, SWAP |
| **Multi‑qubit operations** | Phase/oracle, ancilla oracle, Grover diffusion, n‑Hadamard |
| **Quantum Fourier Transform** | Forward & inverse QFT on arbitrary sub‑registers |
| **Phase estimation** | Configurable precision, used by order‑finding |
| **Order finding** | Implements the quantum order‑finding sub‑routine |
| **Shor's algorithm** | End‑to‑end integer factorisation |
| **OpenMP parallelism** | Coarse‑grained parallel execution of phase‑estimation runs |
| **Logging** | Timestamped log files written to `logs/` |

## Build Instructions
```bash
# Build all examples (default target)
make all
```
Compiled binaries are placed in `bin/examples/`.

To build a single example, e.g. Shor's algorithm:
```bash
make bin/examples/shor
```
Clean artefacts:
```bash
make clean
```

## Running the Examples
### Shor's algorithm
```bash
./bin/examples/shor N
```
Where N is the number to factor.

### Phase‑estimation plot
```bash
./bin/examples/plot_phase_estimation [theta]
```
It plots the success rate of the phase estimation for different values of theta, the angle to be estimated.
`theta` defaults to `0.4`. The final result is saved as a PNG file in the logs folder.

### Product matrix test
```bash
./bin/examples/test_prod_mat
```
It tests the custom product matrix used in the order‑finding circuit.

## Creating Your Own Circuit
1. Include the core headers:
```c
#include "builder/registers.h"
#include "simulator/operations.h"
#include "utils/utils.h"
```
2. Create a quantum register, apply gates, and optionally measure:
```c
QuantumRegister *qreg = qregister_create(3);
apply_gate_hadamard(qreg, 0);
apply_cnot(qreg, 0, 1);
qregister_print(qreg);
```
3. Place the source file under `examples/` and `make all` – it will be compiled automatically.

## API Overview
### Quantum registers (`builder/registers.h`)
| Function | Description |
|---|---|
| `qregister_create(nb_qbits)` | Allocate a register initialised to \|0…0⟩ |
| `qregister_free(qreg)` | Free memory |
| `qregister_set_number(qreg, j)` | Set state to computational basis \|j⟩ |
| `qregister_fuse(q1, q2)` | Tensor‑product of two registers |
| `qregister_measure(qreg, qbit, creg, slot)` | Measure a qubit, collapse state, store result in a classical register |
| `qregister_print(qreg)` | Print amplitudes (limited to ≤4 qubits) |

### Gates (`simulator/operations.h`)
| Function | Description |
|---|---|
| `apply_gate_hadamard(qreg, q)` | Hadamard on qubit *q* |
| `apply_gate_x/y/z(qreg, q)` | Pauli X, Y, Z |
| `apply_gate_phase(qreg, q, φ)` | Phase‑shift Rz(φ) |
| `apply_cnot(qreg, ctrl, tgt)` | Controlled‑NOT |
| `apply_controlled_rotation(qreg, c, t, φ)` | Controlled phase rotation |
| `apply_swap(qreg, q1, q2)` | SWAP |
| `apply_qft(qreg, start, n)` / `apply_iqft` | (Inverse) QFT |
| `apply_oracle_phase/ancilla` | Oracle helpers |
| `apply_diffusion` | Grover diffusion operator |
